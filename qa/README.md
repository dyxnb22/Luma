# QA Artifacts

This folder holds historical run outputs and screenshots. Do not treat screenshots or old reports here as product spec.

Current acceptance entry point: [docs/QA.md](../docs/QA.md).

## Historical run outputs (reference only)

Timestamped folders and round snapshots from past QA sessions are useful for comparing visual regressions, but they are not authoritative behavior.

When behavior conflicts with old QA output, trust code plus `docs/ENGINEERING.md`, `docs/MODULES.md`, `docs/DECISIONS.md`, and `docs/QA.md`.
